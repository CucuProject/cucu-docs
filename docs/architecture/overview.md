# System Overview

The Cucu platform is a distributed microservices architecture built on NestJS with Apollo Federation 2.

## Service Inventory

| Service | Port | DB Port | Purpose |
|---------|------|---------|---------|
| **gateway** | 3000 | N/A | Apollo Federation gateway, authentication entry point |
| **auth** | 3001 | 9001 | Session management, JWT tokens, refresh tokens |
| **tenants** | 3002 | 9002 | Tenant registry, Platform DB (global, not per-tenant) |
| **users** | 3003 | 9003 | User CRUD, profiles, lookup tables |
| **projects** | 3004 | 9004 | Project management, status lifecycle |
| **milestones** | 3005 | 9005 | Milestone CRUD, status tracking |
| **milestone-to-user** | 3006 | 9006 | N:N relationship: users вҶ” milestones |
| **milestone-to-project** | 3007 | 9007 | N:N relationship: projects вҶ” milestones |
| **group-assignments** | 3008 | 9008 | N:N relationship: users вҶ” groups |
| **project-access** | 3009 | 9009 | Project-level access control |
| **grants** | 3011 | 9011 | Permission system (Groups, Permissions, OperationPermissions) |
| **organization** | 3012 | 9012 | Lookup tables: SeniorityLevel, JobRole, Company |
| **bootstrap** | 3100 | N/A | Seed data initialization |

## Data Flow Patterns

### Request Flow (GraphQL Query)

```
1. Client вҶ’ Gateway: POST /graphql with Bearer token
2. Gateway вҶ’ GlobalAuthGuard: Validate JWT
3. GlobalAuthGuard вҶ’ Auth Service: CHECK_SESSION (RPC)
4. Auth Service вҶ’ Gateway: { isValid, userId, groupIds }
5. Gateway: Add signed headers (x-user-id, x-user-groups, x-gateway-signature)
6. Gateway вҶ’ Subgraph: Forward query with headers
7. Subgraph вҶ’ OperationGuard: Check operation permission
8. Subgraph вҶ’ ScopeGuard: Check scope (if @ScopeCapable)
9. Subgraph вҶ’ Resolver: Execute with field filtering
10. Response flows back through federation
```

### Event Flow (User Deletion)

```
1. Users Service: removeUser(userId) called
2. Users Service вҶ’ MongoDB: Set deletedAt, active=false
3. Users Service вҶ’ Redis: Emit 'USER_DELETED' event
4. Auth Service вҶ’ Listener: Receive USER_DELETED
5. Auth Service: Revoke all sessions for userId
6. MilestoneToUser Service вҶ’ Listener: Receive USER_DELETED
7. MilestoneToUser Service: Delete all assignments for userId
8. GroupAssignments Service вҶ’ Listener: Receive USER_DELETED
9. GroupAssignments Service: Delete all group assignments for userId
```

## Key Invariants

1. **Single Entry Point**: All client requests go through the Gateway
2. **Database Isolation**: Each service owns its database
3. **Signed Headers**: All inter-service calls include HMAC signature
4. **Permission Cache**: 5-minute TTL with instant invalidation
5. **Soft Deletes**: Users are soft-deleted (deletedAt timestamp)
6. **Session-Based Auth**: JWT + refresh tokens + server-side sessions
